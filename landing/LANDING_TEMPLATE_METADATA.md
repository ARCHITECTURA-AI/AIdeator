# AIdeator Landing Metadata Notes

This file documents the finalized route architecture for the AIdeator marketing site.

## Site Positioning

- Product: `AIdeator`
- Core claim: local-first idea validation with investor-grade report artifacts
- Primary audiences: founders, product teams, advisors

## Route Map

- `/` home
- `/features/`
- `/product/`
- `/pricing/`
- `/docs/`
- `/compare/`
- `/about/`
- `/blog/`
- `/changelog/`
- `/waitlist/`
- `/contact/`
- `/signin/`
- `/install/`
- `/privacy/`
- `/terms/`

## Metadata Rules

Each route should keep:

- unique `<title>`
- unique `<meta name="description">`
- canonical URL under `https://aideator.app`
- route-level JSON-LD `WebPage`
- one visible `<h1>`

## Conversion Flow

1. Explain pain and trust boundary.
2. Show structured output (4 report cards + markdown artifact).
3. Clarify mode semantics (Local-only, Hybrid, Cloud-enabled).
4. Close with one primary CTA (early access).
