## Security Policy

### Supported Versions

This repository is pre-1.0 (`v0.x`) and under active development. Security fixes are applied only to the latest `main` branch.

### Reporting a Vulnerability

- **Do not** open public issues for suspected security problems.
- Instead, contact the maintainer privately (e.g. via GitHub security advisories or direct email if listed in the repo).
- Include:
  - a clear description of the issue
  - minimal reproduction steps
  - any logs or stack traces (scrubbed of secrets and private data)

### Expectations

- You will receive an acknowledgement within a reasonable time window.
- A proposed mitigation or fix plan will be discussed before disclosure.
